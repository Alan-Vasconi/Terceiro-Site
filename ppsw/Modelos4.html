Modelo Incremental

O Modelo Incremental como o Modelo Espiral usa características dos Modelos anteriores, fazendo o uso de elementos do Modelo Cascata(aplicado repetidamente) com a filosofia iterativa do Modelo de Prototipação, tendo como objetivo desta forma, trabalhar junto com o cliente/usuário para conseguir descobrir os requisitos que ele deseja e ir adicionando ao produto para que ao final tenha o "quebra cabeça completo".

Seguindo esse pretexto, é possível analisar como o Modelo Espiral tem uma abordagem mais simplista da identificação de requisitos, buscando apenas a entrega o mais rápido possível, sendo refinados os elementos a cada entrega ao usuário.


Etapas do Modelo Incremental


O Modelo Incremental tem poucas características únicas, sendo elas:

  • Cada pedaço é desenvolvido de forma linear(Modelo Cascata).
  • Há uma avaliação por parte do cliente a cada entrega, ou seja, incremento.
  • De acordo com o feedback é desenvolvido uma nova versão melhor do que a antiga e entrega novamente


Vantagens e Desvantagens


Como todos os Modelos até agora, também temos a área das vantagens e desvantagens, sendo elas:

  • Vantagens: Há uma maior participação do cliente por terem diversas entregas, podendo os incrementos usados para diversas funções, como a obtenção dos requisitos dos estágios posteriores e na identificação de bugs.
  • Desvantagens: Pode haver uma má manutenção dos incrementos, através de incrementos muito grandes ou do dificil mapeamento dos incrementos que podem gerar uma degradação da arquitetura do sistema com a adição de novos elementos.
