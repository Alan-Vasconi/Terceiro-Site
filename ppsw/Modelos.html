Modelos de Produção de Software

Os Modelos de Produção de Software é um conceito abstrato e informacional para relacionar as formas de organização de um processo de desenvolvimento de software. Desta maneira, é possível dividir em diversos modelos, sendo alguns deles:


Modelo Cascata



O Modelo Cascata é um dos primeiros a surgirem, desta forma, é possível observar características marcantes dos primeiros modelos desenvolvidos, como por exemplo os seus passos são estritamente sequenciais, sendo separado da seguinte forma:

  1. Inicialmente os primeiros passos são espelhados em outras Engenharias como a elétrica ou civil, no qual inicia-se com o análise de requisitos
  2. Pós isso, temos o projeto, implementação, testes e manutenção. Sob essa óptica, é possível analisar como são processos contínuos que cada um depende do sucesso anterior para ser executado, ou seja, um precisa gerar a documentação do outro para continuar os projetos

Porém, mesmo parecendo simples quando se olha a lista de etapas, ainda é um modelo claro, sendo suas etapas descritas das seguintes formas:

  1. Análise de Requisitos: Esta etapa é caracterizada pela base do produto, sendo de extrema importância obter uma visão essencial do software e estabelecer funções, restrições e objetivos do sistema com o publico alvo.
  2. Projeto de Software: Etapa de colocar todos os requisitos no papel de uma forma compreensível para ser codificado nos passos seguintes, ou seja, uma etapa de organização para elaboração de um futuro projeto englobando tudo requisitado, estabelecendo desta forma: Estrutura de Dados, Arquitetura de Software, Detalhes procedimentais e Caracterização das interfaces.
  3. Implementação de Software: Partindo dos documentos preparados no processo anterior esta é a etapa de colocar tudo em prática, no qual, através de uma linguagem de programação definida é elaborado o software e feito teste unitários
  4. Teste do Software: Ponto importânte para o desenvolvimento, no qual todas as unidades que antes estava individuais são integradas e testadas como um todo, buscando possíveis bugs e vendo se está pronta para uso, assim se estiver é feito a entrega logo em seguida.
  5. Manutenção de Software: Esta é a etapa mais demorada de todas, devido ao fato de ocorrer muitas manutenções por diferentes motivos relatados pelos usuários, necessitando de diversas adaptações, e novas funcionalidades.


Vantagens e Desvantagens

A partir da visão geral do Modelo Cascata, é difícil definir exatamente quais seriam suas vantagens e desvantagens, pois cada um das características depende de qual software está sendo desenvolvido. Porém, de maneira geral temos:

  • Vantagens: Sistemas bem documentados e projetados e a equipe se atualiza rapidamente
  • Desvantagens: O início é mais dificil por usar uma abordada mais ousada pois é dificil estabelecer logo de início os requisitos de forma exata, ademais a versão executavel do software demora para ser apresentada e muitos desenvolvedores ficam ociosos durante os processos por um depender do outro para continuar a sua parte.

Desta forma, é possível concluir que mesmo que o Modelo Cascata tenha muitas fragilidades ele é muito melhor que não seguir nenhum modelo, previnindo desta forma, possíveis Bugs Históricos.
